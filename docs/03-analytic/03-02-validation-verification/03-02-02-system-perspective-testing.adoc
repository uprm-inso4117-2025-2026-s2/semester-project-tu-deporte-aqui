
==== 3.2.2 System Testing Perspectives

===== Purpose

This section describes the testing perspectives currently used to evaluate
Tu Deporte Aquí and to identify areas that require additional stabilization.

Rather than defining future testing plans, this section explains how testing perspectives are currently used
to assess responsiveness, usability, and system
behavior under representative usage conditions.

These perspectives complement validation and verification activities by
focusing on observable interaction with the platform.

===== Testing Perspectives

Testing activities were organized around two complementary perspectives:

* system performance under expected user activity
* usability of information access and interaction

These perspectives were selected because Tu Deporte Aquí depends heavily on
rapid interpretation of changing sports information.

===== Performance Perspective

Performance evaluation focuses on whether users can access information without
unnecessary delay under normal usage conditions.

The objective is not to establish formal benchmark values but to identify
conditions that negatively affect responsiveness and user experience.

Observed evaluation concerns included:

* loading behavior across sports pages
* repeated navigation between standings and teams
* responsiveness during multiple simultaneous requests
* rendering behavior when information updates occur

Representative evaluation situations included:

[cols="2,3"]
|===
|Scenario |Observed Focus

|Users opening standings repeatedly
|Page responsiveness and loading behavior

|Multiple users accessing athlete pages
|Concurrent request handling

|Navigation between teams and schedules
|Route transition performance

|Opening pages with incomplete data
|Graceful degradation and recovery
|===

These evaluations helped identify situations that could reduce usability or
system reliability.

===== Usability Perspective

Usability evaluation focuses on whether users can locate and interpret sports
information efficiently.

The evaluation emphasizes practical interaction rather than interface
appearance alone.

Observed usability concerns included:

* discoverability of information
* clarity of page organization
* navigation consistency
* interpretation of incomplete information

Representative evaluation situations included:

[cols="2,3"]
|===
|Scenario |Observed Focus

|Finding standings for a selected team
|Navigation clarity

|Viewing upcoming schedules
|Information accessibility

|Accessing athlete information
|Content organization

|Moving across teams, standings, and news
|Interaction consistency
|===

Special attention was given to minimizing interpretation effort during
time-sensitive sports situations.

===== Relationship to Reliability Goals

System testing perspectives support the broader goals of the project.

Performance contributes to:

* timely access to information
* reduction of interaction delays
* dependable platform behavior

Usability contributes to:

* transparency
* lower interpretation effort
* improved user trust

These perspectives reinforce the project objective of helping users understand
sports information rather than simply displaying data.

===== Current Limitations

Testing activities remain exploratory and reflect the current state of project
development.

Current limitations include:

* incomplete feature implementation
* limited measurement-based performance evaluation
* partial coverage of user interaction scenarios
* evolving interface behavior

These limitations define the current confidence boundary rather than indicate
failure.

===== Summary

System testing perspectives provide a structured way to evaluate how users
interact with Tu Deporte Aquí and how reliably information can be accessed and
interpreted.

The goal of these evaluations is to support iterative improvement rather than
to claim complete verification of the final system.
