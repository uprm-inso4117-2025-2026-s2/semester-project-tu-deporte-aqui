
=== 3.1 Concept Analysis

===== Purpose

This section analyzes the conceptual structures that emerge from the observed
sports information environment described in the descriptive part.

Rather than introducing implementation details, this section identifies and
standardizes the concepts needed to describe how sports information is created,
distributed, interpreted, and trusted inside Tu Deporte Aquí.

The analysis focuses particularly on uncertainty, conflicting reports, temporal
variation, and differences between information sources.

===== From Domain Observations to Concepts

The domain rough sketch identified recurring situations in local Puerto Rico
sports coverage.

Observed situations included:

* identical events being reported by different sources
* delays between event occurrence and publication
* conflicting score reports
* corrections appearing after initial publication
* information becoming unavailable over time
* incomplete game status updates

These observations are transformed into stable concepts that can later be used
consistently throughout requirements, implementation, and validation.

|===
|Observed Situation |Resulting Concept

|Multiple pages describe the same event differently
|Information Conflict

|Updates appear at different times
|Freshness

|Original information source matters

|Information changes after publication

|Users must judge credibility

|Information may disappear or become outdated
|Staleness
|===

===== Concept Definitions

====== Sports Information

Sports Information refers to structured information describing sporting events,
including schedules, scores, standings, game states, announcements, and related
context.

Sports information is treated as a representation of observed events rather
than as guaranteed objective truth.

====== Data Source

A Data Source is any origin from which sports information becomes available.

Examples include:

* official league pages
* school athletic accounts
* social media publications
* reporters and journalists
* livestream updates
* community reports

Different sources may vary in authority, update frequency, completeness, and
consistency.

====== Reliability State

Reliability State represents the current interpretation of how dependable a
piece of information appears within its available context.

Possible states include:

* Confirmed — multiple indicators support consistency
* Provisional — information exists but remains unsettled
* Conflicting — multiple incompatible reports exist
* Missing — expected information is unavailable
* Stale — information has exceeded expected update expectations

Reliability State is not absolute truth but a representation of current
confidence.

====== Freshness

Freshness describes how recently information was known to be updated.

Freshness exists independently from correctness.

Recent information may still be inaccurate, while older information may remain
correct.

====== Traceability

Traceability describes the ability to understand:

* where information originated
* when information was obtained
* whether later corrections occurred
* how the current representation evolved

Traceability supports interpretation under uncertainty.

===== Concept Relationships

The concepts identified in this section interact rather than operate
independently.

[cols="1,3"]
|===
|Concept |Relationship

|Data Source
|Produces Sports Information

|Sports Information
|Receives Reliability State

|Timestamp
|Supports Freshness evaluation

|Freshness
|Influences Reliability State

|Traceability
|Explains information changes

|Reliability State
|Supports user interpretation
|===

Information therefore becomes interpretable not only through content itself,
but through surrounding context.

===== Reliability Interpretation

Reliability is treated as an interpretive concept rather than a guarantee of
correctness.

Two reports may describe the same event differently while both remain visible
to users.

The system therefore emphasizes communication of uncertainty instead of hiding
ambiguity.

This approach supports informed interpretation rather than automated authority.

===== User Trust and Transparency

Trust is influenced not only by whether information is correct, but also by
whether users understand the conditions under which information was obtained.

Users are therefore expected to benefit from visibility of:

* source attribution
* update timing
* conflicting reports
* correction history
* reliability indicators

Transparency supports confidence even when certainty cannot be guaranteed.

===== Conceptual Constraints

The conceptual model operates under constraints observed in the domain.

These constraints include:

* lack of standardized publication formats
* inconsistent reporting schedules
* absence of official APIs
* temporary information availability
* manual update processes

The conceptual model therefore prioritizes interpretation and transparency over
complete automation.

===== Mobile-First Considerations

Since most users are expected to access the platform through mobile devices,
concept presentation must remain concise and quickly interpretable.

Concept communication should emphasize:

* minimal visual overload
* compact status indicators
* immediate understanding of reliability context

===== Summary

Concept Analysis transforms observed domain situations into reusable concepts
that guide requirements, implementation decisions, and later validation
activities.

The resulting concepts provide a shared language for describing uncertainty,
source behavior, information reliability, and interpretation inside Tu Deporte
Aquí.
