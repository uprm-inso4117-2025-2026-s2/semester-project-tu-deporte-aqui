==== 2.1.5 Events, Actions, and Composite Behaviors

This section describes domain-relevant events, actions, and composite behaviors related to how sports information becomes available, changes over time, conflicts with other reports, and remains interpretable under uncertain conditions.

====== Domain-Relevant Events

*Report Arrival*::
A source makes new information about a sporting event available.
Examples include publication of a score update, standings change, postponement notice, or status update.

*Conflict Emergence*::
A newly arrived source report introduces information that is incompatible with information already associated with the same sporting event.

*Correction Arrival*::
A source communicates that previously reported information should be revised, replaced, or clarified.

*Freshness Loss*::
A maintained event record remains without sufficiently recent dependable updates long enough that its currentness becomes questionable for its context.

*Source Interruption*::
A source that had been providing expected updates ceases communication, becomes unreachable, or stops publishing information.

====== Actions

*Record Report*::
Consumes:
* source
* source report
* sporting event reference

Produces:
* recorded source report
* associated report content
* timestamps and attribution references when available

Purpose::
To preserve incoming information for later interpretation, comparison, and traceability.

---

*Assess Dependability*::
Consumes:
* recorded source report
* source trust level
* other related reports
* timing and contextual information

Produces:
* verification state
* confidence level
* possible provisional condition

Purpose::
To evaluate how dependable currently available information appears within its context.

---

*Compare Reports*::
Consumes:
* multiple recorded source reports regarding the same sporting event

Produces:
* consistency finding
* or conflict identification

Purpose::
To determine whether available reports can be treated as mutually supportive or whether incompatibilities remain unresolved.

---

*Update Event Record*::
Consumes:
* assessed report
* current event record

Produces:
* revised event record
* updated timestamps
* updated dependability condition

Purpose::
To revise the maintained representation of what is currently known about the sporting event.

---

*Mark Unknown*::
Consumes:
* event record lacking sufficiently dependable information regarding a relevant value

Produces:
* explicit unknown indication

Purpose::
To avoid presentation of unsupported assumptions or misleading certainty.

---

*Preserve Change Trace*::
Consumes:
* replaced information
* replacement information
* available explanation for the change

Produces:
* traceable record of the revision

Purpose::
To preserve explainability when information changes through correction, reconciliation, or later confirmation.

====== Composite Behaviors

*Information Consolidation*::
This behavior combines:

* report arrival
* record report
* assess dependability
* update event record

It describes how incoming information becomes incorporated into the maintained understanding of a sporting event.

---

*Conflict Handling*::
This behavior combines:

* conflict emergence
* compare reports
* assess dependability
* preserve change trace
* update event record or mark unknown

It describes how incompatible information is managed without silently collapsing differences.

---

*Correction Handling*::
This behavior combines:

* correction arrival
* assess dependability
* preserve change trace
* update event record

It describes how previously communicated information may later be revised while preserving interpretability.

---

*Freshness Management*::
This behavior combines:

* freshness loss
* assess dependability
* mark unknown or retain older information with reduced confidence

It describes how information changes condition when dependable updates stop arriving.

---

*Source Failure Handling*::
This behavior combines:

* source interruption
* freshness loss
* assess dependability
* continue operation under degraded conditions

It describes how interpretation continues when expected information flows become impaired.

====== Remarks

These concepts belong to the domain description itself rather than to a specific implementation strategy.

Later sections may define requirements concerning how systems present, preserve, or process these concepts, but the concepts themselves describe behaviors already present within the domain.
