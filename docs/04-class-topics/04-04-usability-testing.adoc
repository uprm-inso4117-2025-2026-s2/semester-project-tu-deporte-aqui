
=== 4.4 Usability Testing

===== Purpose

This section applies usability-focused testing to Tu Deporte Aquí by examining how easily users can navigate the platform and access the sports information that matters most. Since the project centers on quick access to live games, standings, and news, usability testing helps determine whether those flows are clear, efficient, and understandable.

===== Role of Usability Testing in the Project

Usability testing complements reliability-oriented testing by focusing on the user's experience when interacting with the interface. Even if data is correct, the platform still fails its goal if users cannot quickly find live scores, standings, or the latest sports updates.

Key benefits include:

* Validation of core navigation paths
* Identification of confusing labels or interaction patterns
* Detection of friction when locating high-priority sports information
* Stronger alignment between interface design and real user expectations

===== Representative User Tasks

The following tasks were selected because they reflect the primary value of the platform.

====== Task 1: Find Live Games

*Goal*: A user wants to quickly see whether there are live games and view current score information.

*Expected path*:

. Open the home page.
. Locate the live games section or the live games navigation link.
. Identify a live game and understand its current state.

====== Task 2: Find Standings for a Selected Sport

*Goal*: A user wants to view standings for a preferred sport, such as basketball or baseball.

*Expected path*:

. Open the home page.
. Scroll to the standings section or use the standings navigation link.
. Select the desired sport tab.
. Review the ranking information.

====== Task 3: Find the Latest Sports News

*Goal*: A user wants to access recent sports news updates.

*Expected path*:

. Open the home page.
. Locate the sports news section or the news navigation link.
. Open or review the latest visible news item.

====== Task 4: Determine Whether Information Is Current, Live, or Final

*Goal*: A user wants to understand the state of displayed information without guessing.

*Expected path*:

. View a game card or sports update.
. Use labels, badges, timestamps, or contextual cues to determine whether the information is live, current, or final.

===== Usability Walkthrough Results

The tasks above were evaluated through a simulated usability walkthrough based on the current interface structure and navigation flows.

====== Task 1 Findings: Live Games

*Outcome*: The task can be completed successfully.

*Observations*:

* The top navigation includes a clear "Live Games" entry.
* The home page also highlights live game information in a dedicated section.
* Score and time information are visually prominent enough to support quick scanning.

*Confusion points or barriers*:

* Users may rely on the "LIVE" badge, but there is less clarity for games that are no longer live.
* The distinction between active games and completed games is not always equally explicit.

====== Task 2 Findings: Standings

*Outcome*: The task can be completed successfully with low to moderate effort.

*Observations*:

* The standings section is visible on the home page.
* Sport tabs make it possible to switch between standings categories.
* Ranking information is presented in a compact way that is easy to scan.

*Confusion points or barriers*:

* It may not be immediately obvious whether standings rows are interactive or only informational.
* Users unfamiliar with icons or trend indicators may need stronger labeling.

====== Task 3 Findings: Sports News

*Outcome*: The task can be completed successfully.

*Observations*:

* The sports news section is easy to identify from the home page.
* News cards make it simple to scan titles and short summaries.
* The presence of category labels improves content recognition.

*Confusion points or barriers*:

* Placeholder images or generic visual blocks reduce clarity and polish.
* It may not always be obvious which story is newest if multiple cards look equally prominent.

====== Task 4 Findings: Live vs Current vs Final State

*Outcome*: The task is only partially supported.

*Observations*:

* The interface communicates live content reasonably well through visible status cues.
* Current sports information is grouped into familiar sections, which helps orientation.

*Confusion points or barriers*:

* Final or completed states are not always labeled as explicitly as live states.
* Users may infer status from context rather than from direct language.
* Information freshness could be clearer when a score or update has not changed recently.

===== Improvement Suggestions

Based on the walkthrough, the following improvements would strengthen usability:

* Add a more explicit "FINAL" or completed-state label for finished games.
* Improve visual affordances for clickable rows, cards, or sections.
* Replace placeholder imagery with more meaningful visuals where possible.
* Make recency or freshness indicators easier to interpret.
* Keep badge usage consistent so users can distinguish news categories from game-state labels.

===== Alignment with Project Goals

These usability findings support the broader project goal of making Puerto Rico sports information easy to access and understand. A reliability-focused platform must also be usable under quick, real-world conditions, especially when users open the site to check scores, standings, or updates in only a few moments.

===== Conclusion

Usability testing shows that Tu Deporte Aquí already supports its core information flows reasonably well, especially for live games, standings, and news discovery. However, the walkthrough also highlights opportunities to improve status clarity, navigation affordances, and information freshness cues so users can interpret the platform more confidently and with less effort.
