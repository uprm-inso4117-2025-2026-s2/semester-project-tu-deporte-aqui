
==== 1.2.2 Needs

===== Purpose

This section states the concrete needs that follow from the current situation in Section 1.2.1. The focus is on why Tu Deporte Aquí is needed in the local Puerto Rico sports context, before defining solution details.

===== Context of Need

Local sports information in Puerto Rico is commonly distributed across league pages, team accounts, reporter posts, livestream comments, and community reposts. The same game may appear with different timing, different detail, or different reported values depending on the source consulted.

Because of this, people must repeatedly cross-check information instead of reading one dependable record. The platform need is therefore not generic "sports convenience"; it is the specific need to reduce fragmentation and make reliability visible for local Puerto Rico coverage.

===== User Needs

Fast access from mobile devices::
Fans, families, players, and coaches need to check scores, schedules, standings, and game status quickly from phones during live events, commutes, and school/work breaks.

Reduced cross-platform searching::
Users need one place to consult local Puerto Rico updates without jumping between Instagram stories, Facebook images, league pages, and chat reposts to reconstruct the same event.

Clear status visibility::
Users need immediate clarity on whether information is current, missing, disputed, or corrected so they do not mistake uncertain updates for settled results.

Usable information during high-interest moments::
During playoffs and tournaments, users need dependable updates quickly because delays and conflicts have higher impact on decisions and discussions.

===== System Needs

Aggregation across heterogeneous sources::
The system needs to gather updates from sources that differ in format, timing, and authority without collapsing those differences into one opaque value.

Handling delayed and partial updates::
The system needs to continue operating when some sources are late, incomplete, or temporarily unavailable, while still presenting available information.

Support for revision history::
The system needs to preserve corrections and update history so later changes are visible instead of silently overwriting prior reported values.

Operational transparency::
The system needs to communicate data condition explicitly (for example, partial availability or unresolved conflict) so users understand what is known and what is still uncertain.

===== Data Reliability Needs

Source attribution::
Each displayed value needs clear source identification so users can see where it originated.

Freshness visibility::
Each value needs clear timing context (publish/updated times) so users can judge whether it is still current for its use.

Conflict visibility::
When sources disagree, the disagreement needs to remain visible and understandable, rather than being hidden behind a single unqualified value.

Correction traceability::
When a value changes, users need to know that a correction occurred and what changed.

Qualified confidence::
Information needs to be presented with explicit dependability context so users can distinguish confirmed values from provisional or uncertain ones.

===== Why This Matters for Tu Deporte Aquí

Tu Deporte Aquí is motivated by this combined need profile: local users need faster and clearer access, the system needs to integrate fragmented reporting conditions, and reliability context needs to be visible by default. These needs directly shape the ideas in Section 1.2.3 and later requirements in Section 2.

===== Assumptions

The following assumptions are consistent with Section 1.2.1 observations and project research:
- Local Puerto Rico sports information is often encountered through a mix of official and unofficial channels rather than through one uniform publication mechanism.
- The timing, structure, and completeness of updates vary noticeably between leagues and sources.
- Corrections and late revisions are possible, especially when reporting depends on manual publication or informal communication channels.
- Users in different roles (fan, coach, reporter, organizer) share a need for dependable information, but not all need the same level of detail at the same moment.

===== Scope of This Section

This section defines needs and motivation only. It does not define implementation details or formal requirements. Those appear in Section 1.2.3 (ideas) and Section 2 (requirements).
