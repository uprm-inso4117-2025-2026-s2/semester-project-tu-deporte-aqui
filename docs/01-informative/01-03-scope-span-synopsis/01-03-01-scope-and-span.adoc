==== 1.3.1 Scope and Span

===== Purpose

This section defines the boundaries, coverage limits, and operational constraints of Tu Deporte Aquí. Clearly defining scope prevents scope creep, keeps teams aligned, and establishes realistic expectations for final delivery.

The project has moved from implementation into final integration and stabilization. The scope now focuses on completing the existing product, fixing issues, improving consistency, and preparing the application for the final presentation.

===== In-Scope Coverage

The system scope includes:

===== Local Puerto Rico Sports Leagues

- Coverage is limited to local leagues within Puerto Rico.
- National and international leagues are excluded.
- Emphasis remains on leagues where official APIs may not exist.

This constraint reflects the availability and reliability of data sources.

===== Data Handling Under Uncertainty

The system must handle sports information that may be incomplete, delayed, or inconsistent.

This includes:

- Delayed updates
- Conflicting reports
- Missing information
- Corrected or revised results

In Milestone 3, the focus is on making sure these cases do not break the existing user interface or produce confusing output.

===== Mobile-First Platform

The system is implemented as a mobile-first web platform.

This includes:

- Compact information display
- Clear status indicators
- Prioritization of essential game data
- Responsive layout across devices

For the final milestone, mobile-first scope means fixing layout problems and improving consistency across existing pages rather than introducing new major UI patterns.

===== Final Product Scope

The current milestone includes finalizing and stabilizing the existing product.

This includes:

- Existing UI pages built with Next.js
- Existing backend API routes
- Supabase database and authentication integration
- Games, standings, news, teams, athletes, and authentication pages
- Loading and error states
- Responsive layout behavior
- Final documentation and demo readiness

This defines the transition from building a working application to preparing a stable final product.

===== Out-of-Scope Elements

The following elements remain out of scope:

- Real-time data updates, such as polling or WebSockets
- Automated data scraping mechanisms
- National or international league coverage
- Advanced performance optimization
- Complex user roles or admin systems
- Major database schema redesigns
- New large feature areas not already started in previous milestones

These exclusions limit system complexity and keep the focus on stability, completion, and final presentation readiness.

===== Operational Constraints

Tu Deporte Aquí operates under the following constraints:

===== Informal Data Sources

- Data may originate from websites, reporters, social media, or manually inserted project data.
- Sources may conflict or provide delayed updates.
- Official verification mechanisms may not exist.

Because of this, the system should avoid presenting uncertain information as fully official when that cannot be supported.

===== Limited Structure and Standardization

- Local leagues may lack standardized reporting formats.
- Inconsistent update timing must be expected.
- Historical corrections may occur without formal notice.

These limitations affect how data is displayed and how much confidence users should place in each update.

===== Stability and Integration Requirement

Because this is the final milestone, the system must prioritize stability over expansion.

This means:

- Main pages should not crash during normal use.
- API routes should return valid data.
- Supabase integration should remain stable.
- Authentication should work after refresh and navigation.
- Loading and error states should be visible.
- Major console or runtime errors should be resolved.

These constraints affect how teams decide what to fix, what to polish, and what to leave out.

===== Transparency Requirement

Because of data uncertainty, the system must:

- Communicate provisional or missing information clearly
- Display last update times where applicable
- Indicate possible staleness when data is incomplete
- Avoid implying that unavailable data has been verified

These constraints affect how data is presented and interpreted in the final product.

==== Boundary Summary

Tu Deporte Aquí is a system focused on:

- Local Puerto Rico sports
- Existing pages and routes
- Supabase-backed data display
- Basic authentication
- Mobile-first usability
- Handling unreliable or incomplete sports data
- Final integration, stability, and demo readiness

The project is not focused on adding major new systems in the final milestone.

These boundaries guide final development decisions and help ensure the team completes a stable, understandable product for presentation.
