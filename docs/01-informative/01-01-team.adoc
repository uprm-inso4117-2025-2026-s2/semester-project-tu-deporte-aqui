=== 1.1 Team

===== Project Organization

Tu Deporte Aquí is organized into five specialized teams that support documentation, implementation, integration, and final delivery across project milestones.

During Milestone 1, the focus was documentation and system definition. During Milestone 2, the project moved into implementation. In Milestone 3, the focus shifts to integration, stability, bug fixing, UI polish, and preparing the product for the final presentation.

Each team is assigned a specific domain area. This keeps ownership clear while still requiring coordination across teams, since the final product depends on shared data, API routes, authentication, and common UI behavior.

===== Roles and Responsibilities

====== Developers

Developers are responsible for:

- Completing assigned issues
- Fixing bugs or improving assigned features
- Updating documentation when required
- Testing their own pages, routes, or components
- Submitting work through pull requests

All work must be tracked through GitHub issues.

====== Team Leads

Team Leads are responsible for:

- Coordinating work within their team
- Reviewing work before it is merged
- Ensuring tasks match Milestone 3 goals
- Communicating blockers or dependencies with other teams
- Helping identify unfinished or unstable work before the final deadline

This role helps reduce integration problems between teams.

===== Workflow and Governance

The project follows an issue-driven workflow with pull request review.

Workflow:

1. Issue is created or updated
2. Developer works on a bug fix, documentation update, or feature completion task
3. Changes are submitted through a pull request
4. CI runs to validate the project
5. Team Lead reviews
6. Manager approves and merges

This process helps keep the repository stable while multiple teams complete final integration work.
