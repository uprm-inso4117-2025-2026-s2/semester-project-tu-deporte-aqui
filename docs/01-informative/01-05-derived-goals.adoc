
=== 1.5 Goals

===== Purpose

This section defines the goals that guide Tu Deporte Aquí and connects them to the current situation, stakeholder needs, proposed ideas, and project constraints described earlier in the documentation.

In the previous version, all goals were presented as derived goals. That made it unclear which goals were central to the project and which goals were supporting goals. This revision separates the goals into two groups:

- Primary project goals
- Derived and supporting goals

Primary goals describe the main purpose of Tu Deporte Aquí. Derived and supporting goals describe additional directions that help the project satisfy the primary goals under real constraints, such as incomplete data, informal sources, mobile-first access, and final delivery needs.

===== Primary Project Goals

Primary project goals are the central goals of Tu Deporte Aquí. These goals directly reflect the main reason the project exists.

====== G1 – Centralize Local Puerto Rico Sports Information

Tu Deporte Aquí aims to provide one place where users can access local Puerto Rico sports information.

This includes:

- Games
- Scores
- Standings
- Teams
- Athletes
- News and updates

This is a primary goal because the project is mainly focused on making local sports information easier to find and use.

====== G2 – Make Sports Information Easy to Access on Mobile Devices

Tu Deporte Aquí aims to provide a mobile-first experience for users who check sports information from phones.

This includes:

- Responsive layouts
- Simple navigation
- Compact information display
- Clear page structure
- Readable content on smaller screens

This is a primary goal because users are expected to access scores, standings, and updates quickly during or after games.

====== G3 – Present Sports Information Clearly

Tu Deporte Aquí aims to present sports information in a way users can understand without needing extra explanation.

This includes:

- Clear game status labels
- Organized standings
- Readable team and athlete profiles
- Understandable news or update pages
- Consistent behavior across pages

This is a primary goal because the platform only helps users if the displayed information is clear and usable.

===== Derived and Supporting Goals

Derived and supporting goals are goals that help the project achieve its primary goals. They come from the current situation, system constraints, professor feedback, and the realities of local sports data.

===== Reliability Goals

====== G4 – Handle Incomplete or Conflicting Sports Data

The system should handle sports information that may be incomplete, delayed, conflicting, or corrected after publication.

This includes:

- Missing scores
- Delayed updates
- Conflicting reports
- Corrected results
- Incomplete team or athlete information

This goal is derived from the reality that local sports information may come from informal or inconsistent sources.

====== G5 – Avoid Page Failure When Data Is Missing

The system should remain usable when some data is missing or unavailable.

This includes:

- Showing loading states
- Showing error messages when data cannot be loaded
- Displaying empty states when there is no available content
- Avoiding crashes caused by missing fields

This supports the primary goal of giving users access to sports information even when the available data is limited.

===== Transparency Goals

====== G6 – Communicate Data Limitations Clearly

The system should make data limitations visible instead of hiding them.

This includes:

- Showing when content is unavailable
- Avoiding claims that information is official when it has not been verified
- Indicating when information may be incomplete
- Explaining missing or unavailable data through clear messages

This goal supports user trust because users should understand what the system can and cannot currently provide.

====== G7 – Support Traceability for Important Updates

The system should support traceability when important sports information changes.

This includes:

- Recording when important values change where applicable
- Identifying source or update information when available
- Making corrections understandable
- Avoiding silent replacement of important results

This supports reliability and transparency, especially when scores or standings change after publication.

===== Usability Goals

====== G8 – Keep the Interface Consistent Across Pages

The system should provide a consistent experience across the main application pages.

This includes:

- Consistent navigation
- Similar loading and error states
- Shared layout behavior
- Mobile-friendly page structure
- Visual consistency between feature areas

This goal is derived from the need to make the product usable as one application, not separate pages built in isolation.

====== G9 – Reduce Confusion Around Game and Data Status

The system should make it clear whether information is scheduled, live, final, missing, unavailable, or corrected.

This includes:

- Clear match status labels
- Understandable error messages
- Empty states that explain missing content
- Avoiding conflicting status and score information

This supports both usability and reliability.

===== Scope and Boundary Goals

====== G10 – Maintain Local League Focus

The system should remain focused on local Puerto Rico sports leagues.

This means the project should avoid:

- National league expansion
- International league expansion
- Large new sports coverage areas outside the current scope

This keeps the project realistic and aligned with the original problem area.

====== G11 – Avoid Major New Systems During Final Delivery

The project should avoid introducing major new systems during the final milestone.

This includes avoiding:

- Real-time polling
- WebSockets
- Automated scraping
- Admin dashboards
- Complex user roles
- Major database redesigns

This goal supports final delivery because Milestone 3 is focused on stability, integration, polish, and presentation readiness.

===== Trust and Governance Goals

====== G12 – Keep Documentation Consistent With the Implemented Product

The project should keep documentation aligned with the actual application.

This includes:

- Removing outdated milestone language
- Avoiding unsupported future-tense claims
- Updating documentation based on professor feedback
- Documenting API routes, setup steps, and database information accurately
- Ensuring implementation examples match the real codebase when included

This goal supports the final submission because the documentation should describe the product that actually exists.

====== G13 – Maintain Issue-Based and PR-Driven Collaboration

The team should continue using GitHub Issues and pull requests to organize work.

This includes:

- Tracking work through issues
- Creating branches for assigned tasks
- Reviewing changes before merging
- Using CI checks where applicable
- Keeping responsibility visible across teams

This helps the team coordinate final fixes and documentation updates.

===== Relationship Between Goals

The goals are related, but they are not all at the same level.

The primary goals are:

- Centralize local Puerto Rico sports information
- Make sports information accessible on mobile devices
- Present sports information clearly

The derived and supporting goals explain what the project needs in order to support those primary goals under real constraints.

For example:

- Because local sports data can be incomplete or inconsistent, the system needs reliability and transparency goals.
- Because users are expected to access the platform from phones, the system needs mobile usability goals.
- Because the final milestone focuses on completion, the system needs stability and scope-control goals.
- Because documentation is part of the course deliverable, the project needs documentation and governance goals.

===== Considerations

These goals are not strict requirements. They guide final development decisions and help the team decide what to prioritize.

At this stage, the goals should not introduce major new work. They should support the final product by helping the team stabilize existing features, clarify limitations, improve usability, and keep documentation accurate.

===== Conclusion

Tu Deporte Aquí is primarily focused on making local Puerto Rico sports information easier to find, access, and understand.

The derived and supporting goals describe how the project handles the difficulties around that purpose, including incomplete data, transparency, usability, scope control, documentation, and final delivery.

Separating primary goals from derived goals makes the project direction clearer and addresses the concern that not every goal should be treated as equally derived.
