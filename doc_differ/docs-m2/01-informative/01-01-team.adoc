=== 1.1 Team

===== Project Organization

Tu Deporte Aquí is organized into five specialized teams that support both documentation and implementation across project milestones.

During Milestone 1, the focus was on documentation and system definition. In the current milestone, the project has transitioned into implementation, where teams are responsible for building and integrating system components.

Each team is assigned a specific domain area. This allows work to be distributed while still maintaining coordination across the system.

===== Team Structure

Responsible for maintaining consistency and supporting integration across teams:

- Updating README and project documentation
- Defining folder structure and coding standards
- Documenting API endpoints created by feature teams
- Supporting deployment setup
- Applying professor feedback to documentation

This team does not implement features, but ensures that the work produced by other teams remains consistent and properly documented.

===== Roles and Responsibilities

====== Developers

Developers are responsible for:

- Completing assigned issues
- Implementing features or documentation updates
- Writing code or documentation depending on their team
- Submitting work through pull requests

All work must be tracked through GitHub issues.

====== Team Leads

Team Leads are responsible for:

- Coordinating work within their team
- Reviewing work before it is merged
- Ensuring tasks are completed according to requirements
- Communicating with other teams when needed

This role helps reduce integration issues between teams.

====== Managers

Managers:

- Anthony Martínez
- Fernando Torres

Managers are responsible for:

- Reviewing issues and pull requests
- Assigning urgency and difficulty
- Ensuring milestone progress
- Resolving cross-team issues

Managers ensure that the project stays aligned with milestone goals.

===== Workflow and Governance

The project follows an issue-driven workflow with pull request review.

Workflow:

1. Issue is created and assigned
2. Developer works on a feature or update
3. Changes are submitted through a pull request
4. CI runs to validate the project
5. Team Lead reviews
6. Manager approves and merges

This process ensures that changes are reviewed and integrated consistently.

===== Current Milestone Focus

The project is currently focused on implementation.

This includes:

- Building UI pages using Next.js
- Implementing API routes
- Integrating Supabase
- Connecting frontend and backend
- Implementing authentication

The team structure supports this by distributing responsibility across features while maintaining coordination through shared workflows.
