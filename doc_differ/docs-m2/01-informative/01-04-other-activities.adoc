=== 1.4 Other Activities than Development

===== Purpose

This section describes the non-development activities that support the implementation of Tu Deporte Aquí.

While Milestone 1 focused on research and documentation, the current milestone emphasizes implementation. However, supporting activities such as documentation, coordination, and workflow management remain necessary to ensure consistency and integration across teams.

These activities help maintain structure and reduce errors during development.

===== Documentation-as-Code

The project continues to use a documentation-as-code model using AsciiDoc and GitHub.

This includes:

- Structured folder hierarchy under /docs
- Modular section files linked through include:: directives
- Version-controlled documentation
- Pull-request-based updates
- Automated HTML/PDF builds through CI

Documentation is updated alongside implementation to reflect system behavior and design decisions.

===== Pull Request and Review Process

All contributions are integrated through a PR-based workflow.

The process includes:

1. Issue creation using the standardized template
2. Feature branch creation
3. Implementation or documentation update
4. Pull request submission
5. CI build verification
6. Team Lead review
7. Manager approval and merge

This process ensures that changes are reviewed before being integrated and reduces inconsistencies between teams.

===== CI/CD and Build Automation

The project uses a GitHub Actions workflow that:

- Builds the project and documentation
- Verifies structure and formatting
- Detects errors before merging

This helps prevent broken builds and ensures that changes do not introduce issues.

===== Task Tracking and Governance

All work is tracked through GitHub Issues.

Each issue includes:

- Defined objective
- Acceptance criteria
- Assigned contributor
- Milestone association

Documentation-related issues reference the master scaffold issue (#53) to maintain structure.

This allows work to be distributed while still keeping visibility across teams.

===== Team Coordination

Collaboration is supported through:

- Team-based responsibilities
- Cross-team communication when dependencies exist
- Coordination between frontend, backend, and data teams
- Shared use of components and APIs

Coordination is necessary because features depend on shared data and structure.

===== Milestone Planning and Iteration

The project follows milestone-based development.

This includes:

- Defined implementation goals
- Iterative updates based on feedback
- Adjustments to documentation and structure as the system evolves

Unlike Milestone 1, changes now reflect actual system behavior rather than planned behavior.

===== Conclusion

Tu Deporte Aquí includes activities beyond writing code. Documentation, coordination, and structured workflows support implementation and reduce integration issues.

These activities help ensure that the system is built consistently and that changes remain traceable across milestones.
