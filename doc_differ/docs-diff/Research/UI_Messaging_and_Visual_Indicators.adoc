== [.inserted]#2. UI Patterns for Unverified / Delayed Data#

=== [.inserted]#Pattern 1: Status Badge / Tag#

* Description::
[.inserted]#  Small colored label or chip next to the score indicating reliability (e.g., "Provisional", "Delayed").#

* Use Case::
[.inserted]#  Used when data comes from unconfirmed or secondary sources.#

* Source::
[.inserted]#  https://www.smashingmagazine.com/2025/09/ux-strategies-real-time-dashboards#

---

=== [.inserted]#Pattern 2: Last Updated Timestamp#

* Description::
[.inserted]#  Displays time of last update (e.g., "Updated 2 min ago").#

* Use Case::
[.inserted]#  Helps users understand data freshness and detect delays.#

* Source::
[.inserted]#  https://www.smashingmagazine.com/2025/09/ux-strategies-real-time-dashboards#

---

=== [.inserted]#Pattern 3: Visual State Indicator (Color + Icon)#

* Description::
[.inserted]#  Color-coded indicators to represent data confidence:#

* [.inserted]#Green ✔ → Verified#
* [.inserted]#Yellow ⚠ → Provisional#
* [.inserted]#Red ! → Conflicted#

* Use Case::
[.inserted]#  Allows users to quickly interpret reliability.#

* Source::
[.inserted]#  https://medium.com/design-group-italia/building-user-trust-in-data-d6b943e2c00f#

---

== [.inserted]#3. Conflict Handling Strategy#

* Trigger::
[.inserted]#  When multiple sources report conflicting scores or game data.#

* User Message::
[.inserted]#  "Scores from multiple sources are inconsistent. Awaiting official confirmation."#

* Visual Indicator::
[.inserted]#  Warning icon (yellow/red) + "Conflicted" badge.#

* System Behavior::
* [.inserted]#Freeze displayed score#
* [.inserted]#Continue polling all sources#
* [.inserted]#Update automatically once confirmed#

* Source Insight::
[.inserted]#  https://medium.com/design-group-italia/building-user-trust-in-data-d6b943e2c00f#

---

== [.inserted]#4. Source Attribution Standard (Mobile-First)#

* Placement::
[.inserted]#  Below score in small readable text.#

* Format::
[.inserted]#  "Data Source: @ProviderName"#

* Behavior Rules::
* [.inserted]#Always visible for live games#
* [.inserted]#Append "(Provisional)" if unverified#
* [.inserted]#If multiple sources exist → show primary source only#

* Source Insight::
[.inserted]#  https://medium.com/design-group-italia/building-user-trust-in-data-d6b943e2c00f#
