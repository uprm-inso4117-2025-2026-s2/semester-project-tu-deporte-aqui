== [.inserted]#1. Objective#

[.inserted]#The objective of this research is to design a clear UI system that shows where each news item comes from, allowing users to quickly understand the origin of information.#

== [.inserted]#2. Problem#

[.inserted]#If users cannot determine whether content comes from an official league site, a verified social post, or a data feed, trust decreases.#

[.inserted]#Clear source attribution:#

* [.inserted]#Reduces confusion#
* [.inserted]#Improves credibility#

== [.inserted]#3. UX Principles#

* [.inserted]#*Consistent*: Same badge style across the entire application#
* [.inserted]#*Non-intrusive*: Minimal clutter on the UI card#
* [.inserted]#*Actionable*: Source should be clickable and easily viewable#

== [.inserted]#4. Source Categories#

[.inserted]#Each story should include:#

* [.inserted]#1 primary source category (always shown)#
* [.inserted]#Optional secondary sources (if aggregated)#

=== [.inserted]#A. Official Source#

* [.inserted]#League or team website#
* [.inserted]#Official press release#
* [.inserted]#Official schedule or injury report page#

=== [.inserted]#B. Verified Social#

* [.inserted]#Scraped from verified league or team accounts#

=== [.inserted]#C. API / Structured Feed#

* [.inserted]#Machine-readable provider (scores, stats, schedules)#
* [.inserted]#Example: “Sports Data Feed Provider”#

=== [.inserted]#D. Trusted Media#

* [.inserted]#Editorial news outlet reporting the story#
* [.inserted]#Example: ESPN or local sports media#

=== [.inserted]#E. Community / User Report (Optional)#

* [.inserted]#User-submitted information (non-official, non-media)#

=== [.inserted]#F. Aggregated#

* [.inserted]#Combined data from multiple sources#
* [.inserted]#Must list all underlying sources in details view#

[.inserted]#Place a small badge (icon + label) on the top row of the card, next to time/source metadata.#

* [.inserted]#Always visible#

==== [.inserted]#Cons#
