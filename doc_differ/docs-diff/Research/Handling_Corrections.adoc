---

== [.inserted]#1. Overview#

[.inserted]#Sports information platforms frequently publish data quickly and refine it later.#

[.inserted]#During live games:#

* [.inserted]#Data depends on real-time reporting#
* [.inserted]#Information may be provisional#

[.inserted]#After games:#

* [.inserted]#Leagues may review plays#
* [.inserted]#Corrections may be applied to:#
* [.inserted]#Scores#
* [.inserted]#Player statistics#
* [.inserted]#Outcomes#

[.inserted]#These retroactive corrections are common and necessary for long-term accuracy.#

---

== [.inserted]#2. Problem Context#

[.inserted]#In Tu Deporte Aquí, which aggregates data from multiple Puerto Rican leagues and sources:#

* [.inserted]#Users expect accuracy#
* [.inserted]#Users also expect consistency#

[.inserted]#When data changes after being viewed:#

* [.inserted]#The platform may appear unreliable#
* [.inserted]#Users may assume errors instead of intentional updates#

[.inserted]#Clear communication of corrections is essential.#

---

== [.inserted]#3. Situations Where Corrections Occur#

[.inserted]#Retroactive corrections typically arise from:#

* [.inserted]#Statistical reviews (e.g., video analysis)#
* [.inserted]#Administrative decisions#
* [.inserted]#Post-game official updates#
* [.inserted]#Delayed data from providers#
* [.inserted]#Reinterpretation of statistics (assists, errors, defensive plays)#

---

== [.inserted]#4. Risks to User Trust and Reliability#

[.inserted]#If corrections are not clearly communicated, risks include:#

* [.inserted]#Users seeing different results at different times#
* [.inserted]#Confusion when standings change after being considered final#
* [.inserted]#Reduced confidence in the platform#
* [.inserted]#Temporary inconsistencies across system sections#

IMPORTANT::

[.inserted]#Users must understand that corrections are part of a reliable process—not a system failure.#

---

== [.inserted]#5. Correction Handling in Existing Platforms#

[.inserted]#Sports platforms typically use a combination of:#

=== [.inserted]#Automatic Data Replacement#

* [.inserted]#Updated data replaces old values immediately#

---

=== [.inserted]#Correction Indicators / Timestamps#

* [.inserted]#Show that data has been updated#
* [.inserted]#Helps users understand changes without disruption#

---

=== [.inserted]#Correction Logs / History#

* [.inserted]#Maintain a visible record of changes#
* [.inserted]#Allows verification of updates#

---

== [.inserted]#6. Transparency Mechanisms#

[.inserted]#To maintain trust while keeping the system simple:#

* [.inserted]#Display “last updated” timestamps#
* [.inserted]#Show correction indicators when data changes#
* [.inserted]#Provide optional access to change history#
* [.inserted]#Use clear language#

Example::

[.inserted]#"Stats updated after official review."#

---

== [.inserted]#7. Correction Workflow#

[.inserted]#A structured workflow ensures consistent behavior:#

[.inserted]#1. Detect discrepancies or receive official corrections#
[.inserted]#2. Validate corrections before applying updates#
[.inserted]#3. Update results and dependent data:#
* [.inserted]#Standings#
* [.inserted]#Statistics#
[.inserted]#4. Display correction indicators if visible outcomes change#
[.inserted]#5. Maintain transparency using:#
* [.inserted]#Timestamps#
* [.inserted]#Change history#

---

== [.inserted]#8. Key Design Principle#

[.inserted]#Corrections should:#

* [.inserted]#Improve accuracy#
* [.inserted]#Maintain consistency#
* [.inserted]#Preserve user trust#

[.inserted]#Without:#

* [.inserted]#Overwhelming users#
* [.inserted]#Creating unnecessary alerts#

---

== [.inserted]#Conclusion#

[.inserted]#Retroactive corrections are a normal part of sports data systems.#

[.inserted]#By implementing:#

* [.inserted]#Clear communication#
* [.inserted]#Transparent updates#
* [.inserted]#Consistent workflows#

[.inserted]#Tu Deporte Aquí can:#

* [.inserted]#Maintain accuracy#
* [.inserted]#Reduce confusion#
* [.inserted]#Strengthen user trust#

[.inserted]#even when data changes after publication.#
