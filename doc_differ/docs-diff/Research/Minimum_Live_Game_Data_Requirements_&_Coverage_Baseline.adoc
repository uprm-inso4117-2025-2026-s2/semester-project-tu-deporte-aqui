=== [.inserted]#2.1 Common Live Data Fields#

[.inserted]#After comparing multiple matches across platforms and sports, several fields consistently appeared when a match was labeled as *Live*.#

==== [.inserted]#Core Fields (Always Present)#

* [.inserted]#Team A name#
* [.inserted]#Team B name#
* [.inserted]#League or competition name#
* [.inserted]#Match status (Live, 1H, Q3, HT, etc.)#
* [.inserted]#Score (even if 0–0)#

[.inserted]#These fields appear in both:#

* [.inserted]#List views#
* [.inserted]#Detailed match views#

[.inserted]#IMPORTANT:  #
[.inserted]#If any of these fields were missing, the match was usually *not displayed as live*.#

[.inserted]#NOTE:  #
[.inserted]#The clock may temporarily disappear, but the match can still be labeled *Live* if the score and status are present.#

[.inserted]#Platforms behave differently when live data is incomplete.#

==== [.inserted]#A. Score-Only Live#

[.inserted]#Conditions:#

* [.inserted]#Status = Live#

* [.inserted]#Clock is missing#

[.inserted]#Behavior:#

* [.inserted]#Match remains labeled *Live*#
* [.inserted]#No artificial time is inserted#
* [.inserted]#Clock is simply omitted#

==== [.inserted]#B. Delayed Updates#

[.inserted]#When updates stop arriving:#

* [.inserted]#Some platforms do not explicitly say “delayed”#

[.inserted]#However:#

* [.inserted]#Users can infer delay through timestamps#

== [.inserted]#3. Minimum Live Data Set for MVP#

[.inserted]#A match qualifies as *LIVE* in the MVP if:#

[.inserted]#1. Both teams are identified#
[.inserted]#2. League is identified#
[.inserted]#3. Status indicates in-progress#
[.inserted]#4. Score OR clock is present#
[.inserted]#5. A last updated timestamp exists#

[.inserted]#IMPORTANT:  #
[.inserted]#This ensures that the system never displays *Live* without visible evidence of recency.#
