== [.inserted]#1. Alignment with Milestone 1 Team Goals#

[.inserted]#This research supports the Milestone 1 goal of establishing the project's documentation foundation.#

[.inserted]#The PDF build action enables:#

* [.inserted]#A repeatable process to generate the final documentation as a single PDF#
* [.inserted]#A clean documentation index (e.g., `docs/index.adoc`)#
* [.inserted]#Validation of documentation standards and glossary#
* [.inserted]#Capturing key assumptions in a distributable PDF snapshot#

== [.inserted]#2. Requirements and Constraints#

[.inserted]#The automation must satisfy:#

* Inputs::
* [.inserted]#AsciiDoc files (`*.adoc`)#
* [.inserted]#Assets (images, themes, fonts)#

* Output::
* [.inserted]#Single PDF (optionally multiple PDFs)#
* [.inserted]#Stored as workflow artifact#

* Triggers::
* [.inserted]#Changes to `docs/` or `*.adoc`#
* [.inserted]#Manual execution (`workflow_dispatch`)#

* Reliability::
* [.inserted]#Deterministic builds#
* [.inserted]#Clear failure when inputs are missing#

* Security::
* [.inserted]#Safe mode restricts file access#
* [.inserted]#Includes limited to documentation directory#

* Future-proofing::
* [.inserted]#Use GitHub artifact actions v4+ (v3 deprecated)#

== [.inserted]#5. Recommended Workflow Architecture#

* Docs entrypoint::
[.inserted]#  `docs/index.adoc`#

* Output folder::
[.inserted]#  `build/docs/` (ignored by git)#

* Artifacts::
[.inserted]#  Upload using `actions/upload-artifact@v4`#

* Optional publishing::
[.inserted]#  Use GitHub Releases for versioned PDFs#

---

== [.inserted]#6. Implementation Proposal#

=== [.inserted]#Docker-Based Workflow (Recommended)#

[source,yaml]
----
[.inserted]#name: Docs PDF#
[.inserted]#on:#
[.inserted]#  push:#
[.inserted]#    paths:#
- [.inserted]#"docs/**"#
- [.inserted]#"**/*.adoc"#
- [.inserted]#".github/workflows/docs-pdf.yml"#
[.inserted]#  workflow_dispatch:#

[.inserted]#jobs:#
[.inserted]#  build-pdf:#
[.inserted]#    runs-on: ubuntu-latest#
[.inserted]#    permissions:#
[.inserted]#      contents: read#

[.inserted]#    steps:#
- [.inserted]#name: Checkout#
[.inserted]#        uses: actions/checkout@v4#

- [.inserted]#name: Build PDF (Docker toolchain)#
[.inserted]#        run: |#
[.inserted]#          mkdir -p build/docs#
[.inserted]#          docker run --rm \#
[.inserted]#            -u "$(id -u):$(id -g)" \#
[.inserted]#            -v "${PWD}:/documents" \#
[.inserted]#            asciidoctor/docker-asciidoctor \#
[.inserted]#            asciidoctor-pdf docs/index.adoc -o build/docs/tu-deporte-aqui.pdf#

- [.inserted]#name: Upload PDF artifact#
[.inserted]#        uses: actions/upload-artifact@v4#
[.inserted]#        with:#
[.inserted]#          name: docs-pdf#
[.inserted]#          path: build/docs/tu-deporte-aqui.pdf#
[.inserted]#          retention-days: 14#
----

=== [.inserted]#Ruby-Based Workflow (Alternative)#

[.inserted]#    paths:#

- [.inserted]#"**/*.adoc"#

[.inserted]#jobs:#

[.inserted]#    steps:#

== [.inserted]#7. Documentation Setup Requirements#

[.inserted]#To ensure stable builds:#

* Includes::
* [.inserted]#Keep inside `docs/`#
* [.inserted]#SAFE mode allows includes within directory#

* Images::
* [.inserted]#Store under `docs/images`#
* [.inserted]#Set `:imagesdir: images`#
* [.inserted]#Use `allow-uri-read` for external images#

* Themes::
* [.inserted]#Store in `docs/resources/themes`#
* [.inserted]#Fonts in `docs/resources/fonts`#
