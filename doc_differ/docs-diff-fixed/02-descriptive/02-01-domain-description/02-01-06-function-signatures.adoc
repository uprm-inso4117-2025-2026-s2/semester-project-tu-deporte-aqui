===== Scenario Walkthrough

[.inserted]#A team page posts a final score for a local game. The post is treated as a `SourceReport` because it contains the actual reported score, the source, and the available timing information. The system uses `classifySourcePattern`# to [.removed]#be communicated through visible degraded behavior rather#
[.removed]#than silent omission.# [.inserted]#decide whether the source is acting as a streaming source or a spot source. It then uses `buildReliabilityMetadata` to attach source and timing context to the report.#

[.inserted]#If a league page later posts a different final score for the same event, the second post is also treated as a `SourceReport`. The system uses `detectConflict` to compare both reports against the current `EventRecord`. If the values disagree, the result is a `ConflictAssessment`, and `lowerConfidenceAfterConflict` adjusts the reliability metadata instead of presenting either score as final without explanation.#

[.inserted]#If a later source publishes a correction, `recordCorrection` updates the `EventRecord` and creates a `CorrectionTrace`. The trace keeps the earlier value, the corrected value, the report that caused the change, and the time when the change was recorded.#
