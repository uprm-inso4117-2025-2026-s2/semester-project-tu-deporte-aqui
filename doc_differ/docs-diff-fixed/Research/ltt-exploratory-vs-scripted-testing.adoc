== Introduction

[.inserted]#Software testing includes multiple approaches that help developers and testers evaluate software quality, identify defects, and improve reliability. Two commonly discussed approaches in software testing are scripted testing and exploratory testing.#

[.inserted]#Scripted testing focuses on predefined test cases and expected outcomes, while exploratory testing focuses on active investigation and adaptive learning during the testing process. Both approaches are valuable because they help identify different categories of defects and evaluate software behavior from different perspectives.#

[.inserted]#In sports information platforms, testing becomes especially important because information changes rapidly and users depend on timely and understandable updates. Systems that display schedules, standings, live scores, and status changes must remain reliable even when information is incomplete, delayed, or conflicting.#

[.inserted]#This Lecture Topic Task analyzes exploratory testing and scripted testing, explains their differences, and discusses how both approaches can be applied to Tu Deporte Aquí.#

== Overview of Scripted Testing

[.inserted]#Scripted testing is a structured testing approach in which test cases are designed before execution.#

[.inserted]#A scripted test usually contains:#

- [.inserted]#predefined steps#
- [.inserted]#specific inputs#
- [.inserted]#expected outputs#
- [.inserted]#validation conditions#

[.inserted]#The tester follows the documented instructions exactly in order to verify that the system behaves as expected.#

[.inserted]#Scripted testing is commonly used for:#

- [.inserted]#regression testing#
- [.inserted]#requirement validation#
- [.inserted]#repeated verification of critical features#
- [.inserted]#automated testing workflows#
- [.inserted]#consistency across multiple testing sessions#

[.inserted]#One important advantage of scripted testing is repeatability. Since the steps and expected results are predefined, different testers can execute the same tests and compare results consistently.#

[.inserted]#Another advantage is traceability. Scripted tests can be directly connected to requirements, acceptance criteria, and system behaviors.#

[.inserted]#However, scripted testing also has limitations. Because the tester follows predefined instructions, unexpected defects or unusual interactions may remain undiscovered. Scripted testing tends to focus on expected behavior rather than unpredictable situations.#

== Overview of Exploratory Testing

[.inserted]#Exploratory testing is a flexible testing approach in which learning, investigation, and test execution occur simultaneously.#

[.inserted]#Instead of following rigid predefined test cases, the tester actively interacts with the system while observing behaviors, identifying risks, and adapting testing decisions dynamically.#

[.inserted]#Exploratory testing is useful when:#

- [.inserted]#requirements change frequently#
- [.inserted]#the system contains unpredictable interactions#
- [.inserted]#usability concerns exist#
- [.inserted]#developers want to discover unexpected defects#
- [.inserted]#real user behavior is difficult to predict#

[.inserted]#One major advantage of exploratory testing is adaptability. The tester can immediately investigate suspicious behaviors, unusual outputs, or inconsistent workflows.#

[.inserted]#Exploratory testing is also effective for identifying:#

- [.inserted]#usability problems#
- [.inserted]#navigation issues#
- [.inserted]#confusing interface behavior#
- [.inserted]#inconsistent error handling#
- [.inserted]#unexpected edge cases#

[.inserted]#However, exploratory testing also has limitations. Because testing is less structured, coverage may vary between testers. The results are also harder to reproduce exactly compared to scripted testing.#

[.inserted]#Exploratory testing depends heavily on tester experience, observation skills, and understanding of the system.#
